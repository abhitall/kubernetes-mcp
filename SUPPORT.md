# Support

## Getting Help

Use these channels for the fastest response:

- Questions about setup or usage: open an issue and include logs/config context.
- Bug reports: use the Bug Report issue template.
- Feature requests: use the Feature Request issue template.

## Security Issues

For potential vulnerabilities, do not open a public issue.
Follow `SECURITY.md` for private disclosure.

## What to Include

For technical support requests, include:

- Kubernetes flavor and version
- MCP transport mode (`streamable-http`, `sse`, or `stdio`)
- Relevant configuration (redact secrets)
- Exact error message and logs
- Steps to reproduce
