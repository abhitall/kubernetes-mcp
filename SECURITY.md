# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | Yes |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Preferred reporting channel:

- Use GitHub private vulnerability reporting (Security tab -> Report a
  vulnerability) for this repository.

If private reporting is unavailable, contact the maintainer directly via:

- https://github.com/abhitall
- security@localhost.invalid (replace with your maintainer security mailbox)

Please include:

- A clear description of the vulnerability and impact
- Reproduction steps or proof of concept
- Affected versions/paths
- Suggested remediation (if known)

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Triage target: within 7 calendar days
- Remediation target: based on severity and exploitability

## Disclosure Process

We follow coordinated disclosure. After triage and patch availability, we will
coordinate disclosure timing with reporters when possible.
